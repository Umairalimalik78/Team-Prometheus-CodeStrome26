**4. Database Schema (Supabase / PostgreSQL)**

**4.1 Tables Overview**

|               |                                                   |
|---------------|---------------------------------------------------|
| **Table**     | **Purpose**                                       |
| profiles      | Extended user info linked to Supabase auth.users  |
| habit_modules | Each habit a user is tracking --- generated by AI |
| daily_logs    | User marks daily completion for each habit        |
| reminders     | Scheduled reminder records with tapering phase    |
| checkins      | Day 21-25 AI check-in conversations and decisions |
| chat_messages | Intake interview conversation history per habit   |

**4.2 SQL Schema**

> \-- PROFILES
>
> create table profiles (
>
> id uuid references auth.users on delete cascade primary key,
>
> name text,
>
> timezone text default \'Asia/Karachi\',
>
> created_at timestamptz default now()
>
> );
>
> \-- HABIT MODULES
>
> create table habit_modules (
>
> id uuid default gen_random_uuid() primary key,
>
> user_id uuid references profiles(id) on delete cascade,
>
> habit_name text not null,
>
> goal_description text,
>
> why_it_matters text,
>
> blockers text\[\],
>
> daily_plan jsonb, \-- array of {day, task, tip}
>
> obstacle_playbook jsonb, \-- array of {trigger, response}
>
> env_design_tips text\[\],
>
> phase int default 1, \-- 1=high, 2=moderate, 3=reducing, 4=tapering, 5=autonomous
>
> status text default \'active\', \-- active \| autonomous \| completed \| restarted
>
> streak_count int default 0,
>
> start_date date default current_date,
>
> created_at timestamptz default now()
>
> );
>
> \-- DAILY LOGS
>
> create table daily_logs (
>
> id uuid default gen_random_uuid() primary key,
>
> module_id uuid references habit_modules(id) on delete cascade,
>
> user_id uuid references profiles(id) on delete cascade,
>
> log_date date not null,
>
> completed boolean default false,
>
> note text,
>
> created_at timestamptz default now(),
>
> unique(module_id, log_date)
>
> );
>
> \-- REMINDERS
>
> create table reminders (
>
> id uuid default gen_random_uuid() primary key,
>
> module_id uuid references habit_modules(id) on delete cascade,
>
> user_id uuid references profiles(id) on delete cascade,
>
> scheduled_at timestamptz not null,
>
> content text not null,
>
> phase int, \-- matches habit_modules.phase
>
> delivered boolean default false,
>
> snoozed boolean default false,
>
> snooze_count int default 0,
>
> created_at timestamptz default now()
>
> );
>
> \-- CHECKINS
>
> create table checkins (
>
> id uuid default gen_random_uuid() primary key,
>
> module_id uuid references habit_modules(id) on delete cascade,
>
> user_id uuid references profiles(id) on delete cascade,
>
> checkin_day int, \-- e.g. 21, 25
>
> user_response text,
>
> ai_decision text, \-- \'close\' \| \'extend\' \| \'restart\'
>
> ai_message text,
>
> created_at timestamptz default now()
>
> );
>
> \-- CHAT MESSAGES (intake interview history)
>
> create table chat_messages (
>
> id uuid default gen_random_uuid() primary key,
>
> module_id uuid references habit_modules(id) on delete cascade,
>
> user_id uuid references profiles(id) on delete cascade,
>
> role text not null, \-- \'user\' \| \'assistant\'
>
> content text not null,
>
> created_at timestamptz default now()
>
> );

**4.3 Row Level Security (RLS)**

Enable RLS on all tables. Users can only access their own data:

> alter table habit_modules enable row level security;
>
> create policy \"Users see own habits\" on habit_modules
>
> for all using (auth.uid() = user_id);

Apply same pattern to: profiles, daily_logs, reminders, checkins, chat_messages.

**5. Claude API Integration**

**5.1 Architecture --- Why Supabase Edge Functions**

Claude API key must NEVER be in Flutter code (visible in APK). All Claude calls go through Supabase Edge Functions (Deno runtime). Flutter calls Edge Function → Edge Function calls Claude API → returns result to Flutter.

**5.2 Edge Function 1 --- groq-intake**

Purpose: Powers the AI intake interview. Flutter sends conversation history, gets next AI message back.

> // supabase/functions/groq-intake/index.ts
>
> import { serve } from \'https://deno.land/std/http/server.ts\'
>
> const SYSTEM_PROMPT = \`
>
> You are Habitat, an expert habit coach trained on Atomic Habits,
>
> The Power of Habit, and Tiny Habits. Your job is to interview the
>
> user to understand their habit goal deeply.
>
> Ask these questions ONE AT A TIME (max 6 questions total):
>
> 1\. What habit do you want to build, and why does it matter to you?
>
> 2\. Have you tried building this habit before? What stopped you?
>
> 3\. What are your biggest distractions or obstacles?
>
> 4\. What time of day works best for this habit?
>
> 5\. What does your environment look like --- what helps or hurts?
>
> 6\. On a scale of 1-10, how motivated are you right now?
>
> Be warm, conversational, and encouraging. Ask follow-ups if needed.
>
> When you have enough info, reply with exactly: INTERVIEW_COMPLETE
>
> \`
>
> serve(async (req) =\> {
>
> const { messages } = await req.json()
>
> const response = await fetch(\'https://api.groq.com/openai/v1/chat/completions\', {
>
> method: \'POST\',
>
> headers: {
>
> \'Authorization\': \`Bearer \${Deno.env.get(\'GROQ_API_KEY\')}\`,
>
> \'Content-Type\': \'application/json\'
>
> },
>
> body: JSON.stringify({
>
> model: \'llama-3.3-70b-versatile\',
>
> max_tokens: 1024,
>
> messages
>
> })
>
> })
>
> const data = await response.json()
>
> return new Response(JSON.stringify(data), {
>
> headers: { \'Content-Type\': \'application/json\' }
>
> })
>
> })

**5.3 Edge Function 2 --- groq-generate**

Purpose: Takes full interview conversation and generates the Habit Module JSON.

> // System prompt instructs Claude to return ONLY valid JSON
>
> const GENERATE_PROMPT = \`
>
> Based on the interview conversation, generate a habit module.
>
> Return ONLY valid JSON with this exact structure:
>
> {
>
> \"habit_name\": string,
>
> \"goal_description\": string,
>
> \"why_it_matters\": string,
>
> \"blockers\": string\[\],
>
> \"env_design_tips\": string\[\],
>
> \"daily_plan\": \[{ \"day\": number, \"task\": string, \"tip\": string }\],
>
> \"obstacle_playbook\": \[{ \"trigger\": string, \"response\": string }\],
>
> \"reminder_schedule\": \[
>
> { \"phase\": 1, \"frequency\": \"3x daily\", \"days\": \"1-7\" },
>
> { \"phase\": 2, \"frequency\": \"2x daily\", \"days\": \"8-14\" },
>
> { \"phase\": 3, \"frequency\": \"1x daily\", \"days\": \"15-21\" },
>
> { \"phase\": 4, \"frequency\": \"every 2 days\", \"days\": \"22-25\" }
>
> \],
>
> \"sample_reminders\": \[{ \"phase\": number, \"time\": string, \"message\": string }\]
>
> }
>
> \`

**5.4 Edge Function 3 --- groq-checkin**

Purpose: Evaluates user\'s Day 21-25 progress response and returns a decision.

> // Returns JSON: { decision: \'close\'\|\'extend\'\|\'restart\', message: string }
>
> const CHECKIN_PROMPT = \`
>
> You are evaluating a user\'s habit progress after 21-25 days.
>
> Based on their response, return ONLY JSON:
>
> {
>
> \"decision\": \"close\" \| \"extend\" \| \"restart\",
>
> \"message\": \"encouraging message to the user\",
>
> \"reason\": \"brief explanation\"
>
> }
>
> close = habit is built, no more reminders needed
>
> extend = some progress, extend by 7 more days
>
> restart = not working, restart with easier version
>
> \`

**6. Screens & Navigation Routes**

**6.1 Route Map (go_router)**

|  |  |  |
|----|----|----|
| **Route** | **Screen** | **Description** |
| / | SplashScreen | Check auth state → redirect to /login or /dashboard |
| /login | LoginScreen | Email/password login via Supabase Auth |
| /signup | SignupScreen | New user registration |
| /onboarding | OnboardingScreen | 3-step app intro --- shown only once |
| /dashboard | DashboardScreen | Main screen --- list of active habit modules |
| /chat/:moduleId | ChatScreen | AI intake interview --- new habit flow |
| /habit/:moduleId | HabitDetailScreen | Full habit module detail --- plan, streak, tips |
| /checkin/:moduleId | CheckinScreen | Day 21-25 AI evaluation conversation |

**6.2 Screen Specifications**

**DashboardScreen**

- AppBar: \'Habitat\' logo + profile avatar

- FAB: \'+\' button → navigates to /chat/new (starts new habit interview)

- Body: ListView of HabitCard widgets (one per active habit_module)

- HabitCard shows: habit name, current phase badge, streak count, progress ring (days completed / 25)

- Empty state: illustration + CTA \'Build your first habit\'

**ChatScreen (AI Intake Interview)**

- Full-screen chat UI --- dark background, bubble messages

- AI messages appear with typing indicator (animated dots) before rendering

- User input: TextField at bottom + send button

- When AI replies INTERVIEW_COMPLETE: show \'Generating your habit plan\...\' loader

- Then call groq-generate Edge Function → save to Supabase → navigate to /habit/:moduleId

**HabitDetailScreen**

- Header: habit name + status badge + streak counter

- Progress ring (percent_indicator package) --- days done / 25

- Streak calendar --- 25-day grid, green = completed, grey = missed, white = future

- Today\'s task card --- highlighted daily_plan entry for today

- Environment tips section --- scrollable chips

- Obstacle playbook --- expandable list

- Bottom: \'Mark Today Complete\' button → updates daily_logs + increments streak

**CheckinScreen**

- Triggered automatically when habit reaches day 21

- Chat-style UI with AI asking evaluation questions

- User responds → sent to groq-checkin Edge Function

- Based on decision: update habit_module status → show result screen

**7. Notification System**

**7.1 Why flutter_local_notifications**

For hackathon, flutter_local_notifications is best --- no Firebase setup, works offline, scheduled notifications persist across app restarts on Android.

**7.2 Android Setup Required**

In android/app/src/main/AndroidManifest.xml add:

> \<uses-permission android:name=\"android.permission.RECEIVE_BOOT_COMPLETED\"/\>
>
> \<uses-permission android:name=\"android.permission.SCHEDULE_EXACT_ALARM\"/\>
>
> \<uses-permission android:name=\"android.permission.POST_NOTIFICATIONS\"/\>

**7.3 Reminder Tapering Logic**

|  |  |  |  |  |
|----|----|----|----|----|
| **Phase** | **Days** | **Frequency** | **Times** | **Tone** |
| 1 | 1--7 | 3x daily | 8am, 1pm, 8pm | High energy, tips + environment tricks |
| 2 | 8--14 | 2x daily | 8am, 8pm | Encouraging, reflective |
| 3 | 15--21 | 1x daily | 8am | Autonomy nudges: \'You\'ve got this\' |
| 4 | 22--25 | Every 2 days | 8am | Light check-in tone |
| 5 | 26+ | None | --- | Reminders paused --- check-in triggered |

**7.4 Snooze Logic**

- User can snooze a reminder once (delays by 30 mins)

- If snooze_count \>= 3 on same day → AI sends gentle intervention message next morning

- Snooze count tracked in reminders table
